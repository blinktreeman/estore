Тестовое задание

Необходимо разработать два web-приложения:

  • позволяет управлять деятельностью сети магазинов по продаже электроники
  • приложение-визитка, в котором выводится информация, определенная в функциональных требованиях
Реализация приложений:

Вариант реализации web-интерфейса (frontend) + микросервис с использованием spring-boot (backend)

Приложения обязательно должны иметь web-интерфейс, данные должны храниться в СУБД postgresql

Общие функциональные требования к приложению 1:
  • Хранение основных реестров для добавления, отображения и редактирования информации (список реестров описан в wiki)
  • Хранение вспомогательных справочников (с возможностью редактирования) для идентификации данных в основных реестрах (список справочников описан в wiki)
  • Хранение вспомогательных связующих таблиц для определения множественных ссылок между таблицами (подробнее об этом в структуре БД)
  • Сортировка реестра покупок - по дате осуществления покупки
  • Импорт данных из файлов csv – файлы будут запакованы в zip-архив
  • Проверка на наличие необходимого товара в магазине, в противном случае – запись о покупке не формировать и вывести соответствующую информацию об ошибке во всплывающей подсказке
Общие функциональные требования к приложению 2:
  • Вывод информации о лучших сотрудниках, в зависимости от занимаемой должности. Критерии – количество проданных товаров, на какую сумму продано товаров за последний год.
  • Вывод информации по определенным критериям.
